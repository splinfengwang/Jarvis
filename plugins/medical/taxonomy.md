# Medical Knowledge Taxonomy

> 本模块定义医学领域的术语分类维度。当 medical 插件激活时，扩展知识模型的 L1 术语分类规则。

## 术语分类（L1）

| 类别 | 名称 | 入库位置 |
|---|---|---|
| A | 平台业务领域模型术语 | `知识库/术语/` |
| B | 专业医学知识术语 | `知识库/术语/`，医学标签 |
| C | 外部系统产物 | 不作为独立术语，记录在对应业务域主文档 |

## 判定流程

1. 先问"这是平台自己的概念，还是外部系统的产物？" → C 类入主文档
2. 平台概念入术语体系 → 再问"这是推理需要的基础概念，还是纯医学知识？"
3. 前者 A 类，后者 B 类

## A/B 边界规则

当术语同时具备 A 和 B 属性时，以"推理的主要引用场景"为准：

- 用于理解平台功能/流程/角色 → 按 A 类处理，医学属性作为 tags 标注
- 纯粹用于医学证据检索 → 按 B 类处理

## Tags 规范

- A 类术语必须加业务标签，如 `[术语, 设备]`、`[术语, 指标]`
- B 类加 `[术语, 医学]`
- 禁止 `[术语]` 单标签
