# Obsidian Conventions

## 目的

让 Skill Pack 产物继续符合当前 Obsidian 知识库的组织方式，避免只生成文件但破坏导航和双链。

## 双链

| 类型 | 写法 |
|---|---|
| 术语双链 | `[[术语名]]` |
| Topic 双链 | `[[platform-ops/topics/路径/索引|显示名]]` |
| 业务文档双链 | `[[完整路径|显示名]]` |
| 表格内双链 | 管道符必须转义：`[[路径\|显示名]]` |

不要写悬空双链。目标文件不存在时，先告知林峰；若未创建，将补链事项写入仪表盘跟进事项区。

## 固定载体

| 类型 | 载体 |
|---|---|
| L1 术语 | `知识库/术语/` + `术语索引.md` |
| 知识库主索引 | `知识库/wiki索引.md` |
| 业务分析和知识沉淀 | `业务/<域>/` |
| 医学文献和参考资料 | `知识资产/` |
| 活跃状态 | `platform-ops/仪表盘.md` |
| Topic | `platform-ops/topics/YYYYMMDD_主题简称/` |
| 操作日志 | `platform-ops/log.md` |

## 文件命名

| 文件类型 | 格式 |
|---|---|
| L1 术语 | `<术语名>.md` |
| 业务文档 | `YYYY-MM-DD <域>-<主题>.md` |
| Topic 附加文件 | `YYYY-MM-DD <描述>.md` |
| Topic 骨架 | `索引.md`、`_上下文快照.md`、`_准入检查单.md`、`讨论记录.md` |

已存在文件不强行改名，新文件按规范。

## 仪表盘写入

1. 先读取当前仪表盘。
2. 只改需要改的行，不重写整个表格。
3. 表格内双链管道符必须转义。
4. 写后读回确认列数和格式。

## wiki 页面新建

新建术语、分析文档、决策记录、方案提案等 wiki 页面时：

1. 按命名规范创建文件。
2. 写入 frontmatter。
3. 同步 `知识库/wiki索引.md`。
4. 写入 `platform-ops/log.md`。
