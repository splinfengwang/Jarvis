# Knowledge Model And Ingest

## 四层模型

| 层 | 名称 | 内容 |
|---|---|---|
| L1 | 术语/概念 | 一个业务概念是什么 |
| L2 | 关系与规则 | 概念之间怎么协作，系统怎么运作 |
| L3 | 判断与决策 | 为什么这样设计，选择了什么，放弃了什么 |
| L4 | 待验证/未解决 | 已知但还没答案的问题 |

## L1 术语分类

| 类别 | 名称 | 入库位置 |
|---|---|---|
| A | 平台业务领域模型术语 | `知识库/术语/` |
| B | 专业医学知识术语 | `知识库/术语/`，医学标签 |
| C | 外部系统产物 | 不作为独立术语，记录在对应业务域主文档 |

当术语同时具备 A 和 B 属性，以 Agent 推理的主要引用场景为准。

## 入账触发

对话中出现以下信息时，主动识别并提议入账：

- 新业务术语或概念定义
- 概念之间的关系或业务规则
- 设计决策和判断依据
- 明确提出的待验证问题

入账前向林峰确认。林峰说“先不记”时跳过。

## 新建术语规则

- 新建术语 status 只能是 `待确认`。
- 不存在创建时直接确认的路径。
- 经林峰审核内容后，才可改为 `已确认`。
- 新建术语后必须同步 `术语索引.md` 和 `知识库/wiki索引.md`。

## 执行入库

| 类型 | 处理 |
|---|---|
| L1 术语 | 写入 `知识库/术语/<术语名>.md` |
| L2-L4 | 追加到已有业务文档；无合适文档时新建到 `业务/<域>/` |
| F 文件处理产物 | 写入 `业务/<域>/`，frontmatter 标记来源和质量 |

## frontmatter 最小字段

```yaml
status: 待确认 / 已确认 / 存疑 / 已废弃
source_files: []
source_messages: []
processed_in: [[platform-ops/topics/.../索引|Topic]]
extracted_at: YYYY-MM-DD
evidence_level: 用户确认 / 源文件 / OCR文档解析 / Agent整理 / Agent推断
quality: 描述级 / 对比级 / 归纳级
related: []
```
