# Write Permission

## 目的

防止 Skill Pack 在写文件时降低 `AGENT.md v3.4` 的安全边界。

## 写前检查

写入前先判断：

```text
写入目标：<文件路径>
写入动作：新增 / 修改 / 删除 / 移动 / 批量处理
写入性质：记录性 / 内容性 / 高风险
当前授权：无授权 / 记录授权 / 内容授权 / 高风险授权
是否可执行：是 / 否
原因：...
```

## 分级

| 类型 | 包含动作 | 权限 |
|---|---|---|
| 记录性写入 | 建 Topic、更新快照、追加讨论记录、状态同步、操作日志 | 可自主执行，事后告知 |
| 内容性写入 | 修改分析正文、方案提案、知识条目、业务判断 | 先提案，林峰确认后执行 |
| 高风险写入 | 修改 `AGENT.md`、删除文件、覆盖核心业务文档、批量迁移结构 | 严格审批，先解释影响，再执行 |

只要写入内容会改变未来 Agent 对业务、知识、规则或结论的判断，就不是记录性写入。

## 高风险规则

不能把“继续”“可以”“都行”解释为高风险授权。高风险授权必须明确指向动作和目标文件。

命中任一即高风险：

- 修改 `智能体/贾维斯/AGENT.md`
- 删除文件或目录
- 批量迁移、批量重命名、批量改 frontmatter
- 覆盖核心业务文档
- 修改正式规程、工作流、知识库结构

## 写后验证

以下场景必须读回验证：

- 新建文件
- 编辑已有文档
- 更新仪表盘
- 写入双链或 frontmatter
- 执行脚本修改文件

验证失败时立即修正；无法修正时停止并告知林峰。
