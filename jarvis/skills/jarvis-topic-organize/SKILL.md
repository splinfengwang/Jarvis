# jarvis-topic-organize

## 定位

将已有 Topic（特别是不符合当前规范的旧 Topic）的文件和目录重组为三目录结构：
`参考资料/`、`过程稿/`、`定稿/`。

新 Topic 通过 `jarvis-topic-create` 创建时已自动包含这三个目录，无需调用本 skill。

## 流程

### Step 0: 定位 Topic 并加载规范

1. 定位当前活跃 Topic（读仪表盘）
   → 无活跃 Topic → 告知用户先建 Topic
2. 读取 `jarvis/references/topic-lifecycle.md` 的 `### 文件命名规范` 节

### Step 1: 扫描并分类

列出 Topic 目录下所有**非骨架文件**（排除 `索引.md`、`_上下文快照.md`、`_准入检查单.md`、`讨论记录.md`、`.gitkeep`）。

逐文件判断归属，按以下判据：

| 判据 | 归属 |
|------|------|
| 用户明确标记为最终版/唯一版/确认稿 | `定稿/` |
| 多版本迭代中的被取代旧版；文件名含 `v0.x`/`v1.x` 且不是最高版本 | `过程稿/` |
| PDF/PPT/Word/图片/网页扒取原文等外部输入 | `参考资料/` |
| 同名 `.html` + `.md` 配对出现 | 跟随 `.md` 归类（例如 `架构总览.md` 在 `定稿/` → `架构总览.html` 也归 `定稿/`） |
| 已废弃/过时的独版文件（文件名含 `[废弃]`/`_archived_` 但内容仍有保留价值） | `过程稿/` |

以下归类需标注原因，让用户裁决：

| 情况 | 处理 |
|------|------|
| 单文件无版本号，不确定是过程稿还是定稿 | 列出内容摘要，标注 `⚠️ 需确认归属` |
| 文件内容为讨论中间产物（如"讨论纪要初稿"），但无版本标记 | 建议归 `过程稿/`，标注 `⚠️ 建议` |
| `.xmind`、`.pen` 等非标准格式 | 列出并询问用户意图 |

### Step 2: 目标命名

生成每个文件的目标路径和新名称：

- **定稿/**：`YYYYMMDD-主题-文档类型.md`
  - 日期：取文件原始创建日期或 Topic 讨论中确认该版本的日期
  - 主题：从文件内容提取，不凭空编造
  - 文档类型：方案/BRD/审查报告/设计说明/文案/需求文档 等

- **过程稿/**：`YYYYMMDD-主题-文档类型-vN.M.md`
  - 版本号：从原文件名提取，无法提取则从 v0.1 起编号
  - 同日期多版本：日期不变，版本号递增

- **参考资料/**：保持原名或略加描述前缀

- 二级子目录无例外 — 所有文件都带 `日期-主题` 前缀

### Step 3: 预览重组计划

输出 dry-run 预览表格：

```
### 重组计划

| 当前位置 | → | 目标位置 |
|---------|---|---------|
| 架构总览.md | → | 定稿/20260605-适老化改造-架构总览.md |
| 设计稿/架构总览.html | → | 定稿/20260605-适老化改造-架构总览.html |
| v2.0/需求文档.md | → | 过程稿/20260605-适老化改造-需求文档-v0.2.md |
| ... | → | ... |
```

同时列出：

- **将新建的目录**（如 `参考资料/` 等尚不存在）
- **将空的旧目录**（如 `v1.0/`、`设计稿/` 等，重组后内容已移走）
- **无法自动判断需要用户裁决的项目**

等待用户确认。

### Step 4: 执行重组

用户确认后：

1. 创建尚不存在的标准目录（`参考资料/`、`过程稿/`、`定稿/`）
2. 逐文件执行 `mv`（移动 + 重命名）
3. 清理已空的旧目录
4. 更新 `索引.md` 的 `## 核心文档索引`（三个小节内容按新位置填写）
5. 在 `讨论记录.md` 追加一条重组操作记录

### Step 5: 验证

执行后逐文件验证目标路径存在，列验证结果。

## inputs

- 当前活跃 Topic 的完整目录
- `jarvis/references/topic-lifecycle.md`

## outputs

- 重组计划（dry-run 预览）
- 重组后的 Topic 目录（用户确认后）
- 更新后的 `索引.md`
- 讨论记录追加行

## write_level

- `content_write`：涉及文件移动、重命名、索引更新。必须先 dry-run 预览、用户确认后执行。

## allowed_tools

- `read_file` / `ls` / `glob` — 扫描 Topic 目录
- `bash` — `mv` 移动和重命名文件、`rmdir` 清理空目录
- `write_file` / `edit_file` — 更新 `索引.md` 和 `讨论记录.md`

## confirmation_rules
（通用规则见 JARVIS_CORE §4/§6/§7。以下仅列本 skill 特有规则）
- 用户确认后一次性执行，不逐文件确认

## fallback_rules
（通用规则见 JARVIS_CORE §4/§6/§7。以下仅列本 skill 特有规则）
- 无活跃 Topic → 停止，转 jarvis-topic-create
- Topic 已是标准结构 → 告知无需整理，停止
