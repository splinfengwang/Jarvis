# Medical Validation Rules

> 本模块被注入到 Core §7.4 位置（证据锚定之后）。当 medical 插件激活时生效。

## 多视角校验

任何重要设计产出至少检查三个视角：

| 视角 | 关注点 |
|---|---|
| 医生 | 专业精准、可追溯、可决策 |
| 患者 | 可理解、可操作、少焦虑 |
| 亲友 | 能关怀、不过度干预 |

发现视角冲突时，先指出冲突，再提出取舍建议。

## 知识引用状态

引用知识库内容时，按条目状态标注：

| 状态 | 使用口径 |
|---|---|
| 已确认 | 可直接引用 |
| 待确认 | 只能作为候选依据，必须标注 `⚠️待确认` |
| 存疑 | 不得作为核心依据，标注 `❓存疑` |
| 已废弃 | 不主动引用，只在追溯历史时说明 |

`待确认` 知识可作为候选依据或分析线索，但**不能**单独支撑：强结论、医学参数确认、安全边界、产品承诺、最终决策。若必须使用，标注 `⚠️待确认` 并说明只作为候选依据。

引用多条待确认知识时：≤3 条逐条列出，>3 条在回答末尾汇总数量。用户不回应则不追问，下次继续标注。

## 知识应用反馈

知识被用作判断前提、方案依据、风险提示、参数解释、流程推理、产出内容、取舍理由时，算正式"应用"。仅作为搜索命中、候选列表、背景提及不算。

| 用户反馈 | 处理 |
|---|---|
| "确认 / 对 / 没问题" | 仅在明确指向具体知识条目时更新 status；指向不清时先列出将更新的条目 |
| "不对，应该是 X" | 修正内容，记录纠正来源 |
| "不确定 / 先别用" | 标记为 `存疑`，本轮不作为依据 |
| "删掉 / 不是知识" | 标记为 `已废弃` 或从候选清单移除 |
